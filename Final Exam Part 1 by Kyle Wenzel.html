Client-server application: A client-server application is a software architecture where one computer (client) requests services or resources from another computer (server) over a network. In a client-server application, the server typically provides the resources or services, while the client requests and utilizes them.
Socket programming: Socket programming is a way for computers to communicate with each other over a network by using sockets, which are endpoints for communication. Socket programming allows developers to create client-server applications and transfer data between them using a variety of protocols.
IP address: An IP address is a numerical identifier assigned to every device connected to the internet that uses the Internet Protocol to communicate. IP addresses are used to uniquely identify devices and facilitate communication between them on a network.
TCP port: A TCP port is a number used to identify a specific process to which data is sent when it is received by a device on a network. TCP ports are used by computers to establish connections and send data to specific services or applications running on other devices.
HTTP: HTTP (Hypertext Transfer Protocol) is a protocol used for transferring data over the internet, most commonly used for accessing websites. HTTP operates on top of the TCP/IP protocol, and it defines how clients and servers should communicate with each other to exchange information such as web pages, images, videos, and other types of content.

sessionStorage: sessionStorage is a web storage API that allows web developers to store key-value pairs in a user's browser for the duration of a session. Data stored in sessionStorage is available only to the window or tab that created it, and it is deleted when the window or tab is closed.
localStorage: localStorage is a web storage API that allows web developers to store key-value pairs in a user's browser for an extended period of time. Data stored in localStorage is available across all windows and tabs and persists even after the browser is closed, making it useful for storing user preferences or other data.
IndexedDB: IndexedDB is a client-side web storage API that provides a powerful and flexible way to store and retrieve structured data in a user's browser. With IndexedDB, web developers can create object stores and indexes, and use advanced queries to efficiently retrieve data, making it ideal for building complex web applications that require offline data storage and synchronization.

Backbone: Backbone is a lightweight JavaScript library that provides a simple and flexible way to structure and organize client-side web applications. Backbone uses a Model-View-Controller (MVC) architecture, allowing developers to separate concerns and easily maintain and extend their codebase.
Ember: Ember is a JavaScript framework for building ambitious web applications, with a focus on developer productivity and ease of use. Ember uses a convention-over-configuration approach, providing a set of defaults and conventions that help developers quickly build complex applications.
Underscore: Underscore is a utility library for JavaScript that provides a set of functional programming tools and helpers, such as iterators, arrays, and object manipulation methods. Underscore is designed to be lightweight and flexible, making it easy to integrate into any JavaScript project.
Bootstrap: Bootstrap is a popular front-end framework that provides a set of HTML, CSS, and JavaScript components and tools for building responsive, mobile-first web pages and applications. Bootstrap includes a wide range of components, such as navigation bars, buttons, forms, and modals, making it easy to build consistent and professional-looking interfaces.

CSS is a stylesheet language that uses a plain-text syntax to style web pages. LESS and SASS are both CSS preprocessors that extend the functionality of CSS with additional features and a more advanced syntax. SASS uses a syntax inspired by programming languages like Ruby, while LESS uses a syntax inspired by JavaScript.
LESS and SASS provide several features that are not available in CSS, such as variables, mixins, nested rules, and functions. These features make it easier to write and maintain complex stylesheets, and allow for more efficient reuse of code.
CSS is relatively easy to learn and understand, while LESS and SASS have a steeper learning curve due to their advanced features and syntax.

GCP Cloud Run is a fully managed serverless platform that allows developers to deploy and run containerized applications. It automatically scales the number of instances based on incoming traffic, and charges only for the actual usage.
Firestore is a NoSQL document database service provided by Google Cloud Platform (GCP). It is designed for real-time and scalable applications, allowing developers to store and query data in a flexible and structured way, and offers strong consistency, automatic indexing, and easy integration with other GCP services.
A Bucket in GCP refers to a container for storing and organizing objects, such as files or images, in Google Cloud Storage. Buckets provide durable and highly available storage, and can be configured with fine-grained access control and lifecycle policies.
GKE, or Google Kubernetes Engine, is a fully managed Kubernetes service provided by GCP. It allows developers to deploy, manage, and scale containerized applications in a flexible and efficient way, and integrates seamlessly with other GCP services such as Cloud Load Balancing and Cloud Storage.

NPM (Node Package Manager) is a package manager for Node.js that allows developers to easily install and manage dependencies for their projects. It provides access to a vast collection of open-source packages and simplifies the process of sharing and reusing code.
NPX is a package runner tool that comes bundled with NPM. It allows developers to run a command-line tool or script without having to install it globally or manually manage its dependencies, and can be used for one-off tasks or as part of a development workflow.
NVM (Node Version Manager) is a tool that allows developers to easily switch between different versions of Node.js installed on their system. It provides a simple command-line interface for managing multiple Node.js installations, and is particularly useful for testing and development purposes.
The package.json file is a configuration file for NPM that contains information about a Node.js project, such as its name, version, dependencies, and scripts. It is used to manage project dependencies, specify build scripts, and define other project-specific settings.
The package-lock.json file is a lockfile automatically generated by NPM when installing dependencies for a project. It provides a deterministic and reproducible build by ensuring that all dependencies are installed at the exact version specified in the file, preventing version conflicts and guaranteeing consistent builds across different environments.

Angular uses declarative templating with HTML and custom directives to build UI components, whereas React uses JSX, which allows developers to write HTML-like code directly in JavaScript.
Angular provides built-in state management through its reactive forms and observables, while React relies on external libraries such as Redux or the Context API for state management.
Angular has a more extensive set of tools and features built-in, such as the Angular CLI, which provides a comprehensive set of tools for building, testing, and deploying applications. React has fewer built-in tools, but its modular design allows for more flexibility in choosing external libraries and tools.
Angular has a steeper learning curve than React due to its complex architecture and numerous built-in features. React, on the other hand, has a shallower learning curve because of its simple and modular design.

Class selector: A selector that matches elements based on their class attribute values.

Descendant selector: A selector that matches elements based on their relationship to another element.
Child selector: A selector that matches elements that are direct children of another element.
Pseudo-class selector: A selector that matches elements based on their state or context, such as :hover or :first-child.
Universal selector: A selector that matches all elements.
Adjacent sibling selector: A selector that matches an element that is immediately followed by another element.
Attribute value selector: A selector that matches elements based on the presence or absence of a certain attribute value.

Vanilla JS is plain JavaScript without any third-party libraries or frameworks, while jQuery is a popular JavaScript library that simplifies DOM manipulation, event handling, and AJAX calls. jQuery UI is a collection of user interface widgets, effects, and themes built on top of jQuery, designed to make it easier to create complex UI components.

The Mounting phase is when a component is initialized and inserted into the DOM for the first time, which involves calling its constructor, render() method, and componentDidMount() method.
The Updating phase occurs when a component's state or props change, which involves calling the render() method, componentDidUpdate() method, and shouldComponentUpdate() method to check if an update is needed.
The Unmounting phase occurs when a component is removed from the DOM, which involves calling the componentWillUnmount() method.

useState: The useState hook allows a function component to declare and use state variables, which can be updated and cause a re-render of the component when changed.
useEffect: The useEffect hook allows a function component to perform side effects (such as fetching data or updating the DOM) after rendering, by subscribing to state and props changes.
useContext: The useContext hook allows a function component to consume data and actions from a parent component's context (a global state or theme), without needing to pass down props manually.
useRef: The useRef hook allows a function component to access and modify a mutable value (such as an input field or a timer ID) across renders, without triggering a re-render.
useReducer: The useReducer hook allows a function component to manage complex state with a reducer function (similar to Redux), by dispatching actions that update the state based on the previous state and action type.
useCallback: The useCallback hook allows a function component to memoize (cache) a function instance, preventing it from being re-created on every render, and only changing when its dependencies change.
useMemo: The useMemo hook allows a function component to memoize (cache) the result of a function, preventing it from being re-computed on every render, and only changing when its dependencies change.

Props: They are the data that is passed from the parent component to the child component, and it helps in creating dynamic components that can be reused.
JSX: It is a syntax extension of JavaScript that allows us to write HTML-like code inside JavaScript code, making it easier to create and manage complex UI elements.
RxJS: It is a library that allows us to write reactive programs by providing tools for working with asynchronous data streams.
Virtual DOM: It is a lightweight representation of the actual DOM, used by React to optimize the rendering process. It helps to minimize the number of updates required by the real DOM, which makes the application faster.
